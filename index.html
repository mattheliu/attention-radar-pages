Attention Radar Privacy Policy

Attention Radar is a Chrome extension that shows confirmed public attention signals on supported technical platforms, including GitHub, Hugging Face, Hacker News, X, Reddit, and Bluesky.

Data We Access

The extension may read the current supported page URL, page title, visible page content needed to locate platform controls, and public platform metadata returned by official or public APIs.

Examples include repository owners, public stargazers, public contributors, public commenters, post authors, public likes/reposts/quotes where the platform API exposes actor identities, and public engagement counts.

When a user provides an X API Bearer Token, the token is stored locally in the user's browser and is used only to call official X API endpoints for the current X post being inspected.

When a user submits a high-signal actor suggestion in the popup, the extension stores the suggested profile URL, recommendation reason, detected platform, current page URL/title when available, timestamps, and pending status locally in the browser.

When a user changes radar focus settings, the selected focus pack IDs are stored locally as extension preferences.

Data Storage

All extension data is stored locally in chrome.storage.local.

The extension does not use a backend server in this MVP. It does not sell data, share data with third parties, or run advertising. JSON export is initiated by the user and downloads local extension data to the user's device.

Data Transmission

The extension sends network requests only to the supported public platform APIs and page endpoints required for user-facing features:

  • GitHub REST API
  • Hugging Face public Hub API
  • Hacker News Firebase API
  • X official API, only when the user has locally configured a Bearer Token
  • Reddit public JSON endpoints
  • Bluesky public AppView API

The extension does not transmit the user's stored evaluations, review notes, high-signal actor suggestions, or X token to an Attention Radar backend.

Permissions

The extension uses Chrome permissions for local storage, alarms for review reminders, clipboard writing for user-triggered memo export, and active tab access to prefill a current-page suggestion URL.

User Control

Users can export their local data as JSON from the popup. Users can clear local extension data from the popup. Users can remove the extension at any time from Chrome's extension manager.

Limited Use Statement

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Contact

For beta feedback, use the feedback channel provided to trusted testers or the feedback form linked from the extension listing.

中文

关注雷达隐私政策

关注雷达是一个 Chrome 插件，用来在 GitHub、Hugging Face、Hacker News、X、Reddit、Bluesky 等技术平台页面上显示已经确认的公开关注信号。

我们读取哪些数据

插件可能读取当前支持平台页面的 URL、标题、用于定位页面控件的可见页面内容，以及官方或公开 API 返回的平台公开元数据。

例如：仓库 owner、公开 stargazer、公开 contributor、公开评论者、帖子作者、平台 API 明确暴露身份的点赞/转发/引用账号，以及公开互动计数。

如果用户填写 X API Bearer Token，该 token 只会保存在用户浏览器本地，并只用于请求当前 X 帖子的官方 X API 数据。

如果用户在 popup 中提交“大佬推荐”，插件会在本地保存推荐的主页链接、推荐理由、识别出的平台、当前页面 URL/标题、时间戳和待审核状态。

如果用户调整关注领域设置，插件会把选中的领域包 ID 作为本地偏好保存。

数据存储

所有插件数据都保存在 chrome.storage.local。

这个 MVP 没有后端服务器。插件不会出售数据，不会把数据共享给第三方，也不会投放广告。JSON 导出由用户主动触发，并下载到用户自己的设备。

数据传输

插件只会向实现用户可见功能所需的公开平台 API 或页面接口发起请求：

  • GitHub REST API
  • Hugging Face public Hub API
  • Hacker News Firebase API
  • X official API，仅当用户本地配置 Bearer Token 后使用
  • Reddit public JSON endpoints
  • Bluesky public AppView API

插件不会把用户保存的评估记录、复盘笔记、大佬推荐或 X token 发送到 Attention Radar 后端。

权限

插件使用 Chrome 权限来完成本地存储、复盘提醒、用户主动复制 memo，以及为“大佬推荐”预填当前页面链接。

用户控制

用户可以在 popup 中导出本地 JSON 数据，也可以清空本地数据。用户也可以随时在 Chrome 扩展管理器中卸载插件。

Limited Use 声明

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

联系方式

内测反馈请使用 trusted testers 收到的反馈渠道，或插件商店页提供的反馈表单。
